# school-api

Это REST API для управления школьным расписанием.  
Проект сделан как тестовое задание и позволяет работать с уроками, расписанием, учителями, кабинетами и классами.

## Что умеет проект

- получать список уроков;
- получать урок по id;
- создавать, обновлять и удалять уроки;
- получать общее расписание;
- получать расписание класса на конкретную дату;
- обрабатывать ошибки через `@RestControllerAdvice`;
- показывать документацию через Swagger.
- производить автотестирование через JUnit5 и Mockito

## Стек

- Java 17
- Spring Boot
- Spring Data JPA
- Spring MVC
- Validation
- Flyway
- PostgreSQL
- Swagger / OpenAPI
- JUnit 5
- Mockito

## Запуск проекта

### 1. Поднять базу данных
Если используешь Docker (не забудь проверить порт)
Также добавлен файл наполнения "data.sql":

```bash
docker compose up -d
```

### 2. Запустить приложение

```bash
mvn spring-boot:run
```

Либо просто запустить проект в IDEA.

### 3. Открыть Swagger

После запуска документация доступна здесь:

```bash
http://localhost:8080/swagger-ui/index.html
```

## Обработка ошибок

В проекте настроены основные HTTP-ошибки:

- `400 Bad Request` — ошибки валидации;
- `404 Not Found` — если сущность не найдена;
- `409 Conflict` — если возникает конфликт;
- `500 Internal Server Error` — если случилась непредвиденная ошибка.

## Тесты

Для сервисного слоя написаны юнит-тесты на `JUnit 5` и `Mockito`.

Что проверяется:
- поиск урока по id;
- ошибка, если урок не найден;
- создание урока;
- обновление урока;
- soft delete;
- получение расписания;
- фильтрация расписания по классу и дате.

Запуск тестов:

```bash
mvn test
```

## Структура проекта

- `controller` — REST-контроллеры;
- `service` — бизнес-логика;
- `repository` — работа с базой;
- `entity` — сущности;
- `dto` — объекты для передачи данных;
- `exception` — кастомные ошибки и их обработка.
- `test` — юнит тесты для серверного слоя
